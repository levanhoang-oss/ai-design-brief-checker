# AI Design Brief Checker

A React web application for architecture, urban design, interior design, and landscape architecture students to review whether their concept design responds well to a project brief.

## How to Run Locally

### Prerequisites
- Node.js (version 16 or higher) — download from https://nodejs.org
- npm (comes with Node.js)

### Steps

1. **Download or clone the project folder**

2. **Open Terminal and navigate to the project folder**
   ```bash
   cd path/to/ai-design-brief-checker
   ```

3. **Install dependencies**
   ```bash
   npm install
   ```

4. **Start the development server**
   ```bash
   npm start
   ```

5. **Open the app**
   - It will automatically open at http://localhost:3000
   - If not, open that URL in your browser

### Build for production
```bash
npm run build
```
This creates a `build/` folder you can deploy to any static web host (Netlify, Vercel, GitHub Pages).

## How to Use the App

1. Fill in the **Project Identity** fields (name, type, stage, review focus)
2. Paste your **Design Brief** — the more detail, the better the review
3. Paste your **Concept Statement** — explain how your design responds to the brief
4. Click **Optional strategies & context** to expand fields for site context, stakeholders, sustainability, accessibility, and heritage
5. Click **Generate Review →**
6. Read the structured report in the right-hand panel — navigate using the tabs

## Connecting to the Claude AI API

The app currently runs entirely in the browser using rule-based logic. To connect it to a real Claude AI review:

### Step 1 — Get an API key
Sign up at https://console.anthropic.com and create an API key.

### Step 2 — Create a .env file
In the root of the project, create a file called `.env`:
```
REACT_APP_ANTHROPIC_API_KEY=your_api_key_here
```

### Step 3 — Update reviewEngine.js
In `src/utils/reviewEngine.js`:

1. **Uncomment** the `generateReviewWithClaude()` function (around line 200)
2. **In the `generateReview()` export** at the bottom, replace:
   ```javascript
   return analyseLocally(formData);
   ```
   with:
   ```javascript
   return await generateReviewWithClaude(formData);
   ```
3. **Make `generateReview` async** by adding the `async` keyword:
   ```javascript
   export async function generateReview(formData) {
   ```

### Step 4 — Update App.jsx to await the result
In `src/App.jsx`, update `handleGenerate` to await the async call:
```javascript
async function handleGenerate(data) {
  setIsGenerating(true);
  setFormData(data);
  const result = await generateReview(data);
  setReport(result);
  setIsGenerating(false);
  // ...
}
```

### ⚠️ Security note
Never commit your `.env` file or expose your API key in a public repository. For a real production deployment, route all API calls through a backend server.

## Connection to Part 1 Claude Skill

This web app is the student-facing interface for the same review logic defined in the **design-reviewer** Claude Skill from Part 1.

| Part 1 Skill | Part 2 Web App |
|---|---|
| Triggered by `/design-reviewer` in Claude chat | Launched as a standalone web page |
| Intake form presented as an interactive Claude Artifact | Intake form built as a React component |
| Review generated by Claude's analysis | Review generated by `reviewEngine.js` (rule-based, upgradeable to Claude API) |
| Output displayed as a review dashboard Artifact | Output displayed as a tabbed React dashboard |
| `.docx` export via the docx skill | Extendable with a print/export button |

The `reviewEngine.js` file mirrors the exact same **review categories, scoring rubric, and output structure** defined in the SKILL.md from Part 1 — the same 11 report sections, the same 0–100 weighted scoring system, and the same finding categories (Fully Addressed / Partially Addressed / Not Addressed).

2. **Fill in a quick example** using a made-up project — e.g. a community library concept design. Paste a 2–3 sentence brief and a 2–3 sentence concept statement.

4. **Walk through the report tabs** — point out the score grid, the Brief alignment tab (Fully / Partially / Not addressed), and the Recommendations tab with priority levels.

5. **Show the reviewEngine.js file** — demonstrate the `AI INTEGRATION POINT` comment block. "This is where we'd swap in the Claude API call. The JSON schema returned would be identical — the UI doesn't need to change."
